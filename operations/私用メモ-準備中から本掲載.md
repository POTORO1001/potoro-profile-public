# 私用メモ: プロフィールを本掲載・変更する手順

`maids.json` を編集して、プロフィール準備中のメイドさんを本掲載にしたり、すでに本掲載済みのプロフィール内容を変更したりする手順です。

## 最初に作業場所へ移動

```powershell
cd "C:\Users\infop\OneDrive\ドキュメント\ポトロポータル\potoro-profile-public"
```

作業前にGitHub側の最新を取り込みます。

```powershell
git pull --rebase origin main
```

## ① プロフィール未確定で作成済み → 本掲載に変更するとき

準備中で載っているメイドさんを、本掲載へ切り替える作業です。

### 1. maids.jsonをメモ帳で開く

```powershell
notepad maids.json
```

### 2. 対象メイドさんを探す

例として `うさ` さんなら、次のような部分を探します。

```json
{
  "name": "うさ",
  "birthday": "?/?",
  "generation": "七代目",
  "likes": [
    "準備中"
  ],
  "favorite_food": "準備中",
  "one_liner": "プロフィール準備中です♡",
  "image": "img/profile preparation.png",
  "status": "preparing"
}
```

### 3. 本掲載の内容に変更する

主に変更する項目です。

- `birthday`: 実際の誕生日にする
- `likes`: 好きなものにする
- `favorite_food`: 好きな食べ物にする
- `one_liner`: ひとことにする
- `image`: 本人写真にする場合は画像パスを変更する
- `status`: `"status": "preparing"` の行を削除する

例:

```json
{
  "name": "うさ",
  "birthday": "7/6",
  "generation": "七代目",
  "likes": [
    "YouTube",
    "TikTok"
  ],
  "favorite_food": "焼き鳥、鴨肉",
  "one_liner": "たくさんお話しようね♡",
  "image": "img/usa.JPG"
}
```

本人写真を使う場合は、先に画像を `img/` フォルダへ入れておきます。

### 4. 保存して閉じる

メモ帳で保存して閉じます。

### 5. 検証

```powershell
& 'C:\Users\infop\.cache\codex-runtimes\codex-primary-runtime\dependencies\node\bin\node.exe' scripts\validate-maids.js
```

成功すると、このように表示されます。

```text
maids.json validation passed (19 entries).
```

### 6. GitHubへ反映

```powershell
git status
git add maids.json
git commit -m "Publish maid profile"
git push origin main
```

GitHub Pagesの反映後、ページ上では「準備中」チップが外れて本掲載になります。

## ② すでに本掲載済みのプロフィール内容を変更するとき

本掲載済みのメイドさんの、誕生日・好きなもの・好きな食べ物・ひとこと・画像などを変更する作業です。

### 1. maids.jsonをメモ帳で開く

```powershell
notepad maids.json
```

### 2. 対象メイドさんを探す

例:

```json
{
  "name": "うさ",
  "birthday": "7/6",
  "generation": "七代目",
  "likes": [
    "YouTube",
    "TikTok"
  ],
  "favorite_food": "焼き鳥、鴨肉",
  "one_liner": "たくさんお話しようね♡",
  "image": "img/usa.JPG"
}
```

### 3. 変更したい項目だけ直す

たとえば `one_liner` だけ変えるなら、その行だけ変更します。

```json
"one_liner": "新しいひとことをここに入れる♡",
```

写真を変更する場合は、先に画像を `img/` フォルダへ入れて、`image` を変更します。

```json
"image": "img/new-photo.JPG"
```

### 4. 保存して閉じる

メモ帳で保存して閉じます。

### 5. 検証

```powershell
& 'C:\Users\infop\.cache\codex-runtimes\codex-primary-runtime\dependencies\node\bin\node.exe' scripts\validate-maids.js
```

成功すると、このように表示されます。

```text
maids.json validation passed (19 entries).
```

### 6. GitHubへ反映

```powershell
git status
git add maids.json
git commit -m "Update maid profile"
git push origin main
```

## よくある注意

### 最後の項目にカンマを付けない

次は失敗します。

```json
"image": "img/usa.JPG",
```

最後の項目は、次のようにカンマなしにします。

```json
"image": "img/usa.JPG"
```

### `validate-maids` で失敗したらpushしない

検証で失敗した場合は、`git add` や `git commit` に進まず、エラー内容を確認します。

### `git push origin main` で `fetch first` が出たとき

GitHub側に先に変更が入っています。無理に進めず、まず次を実行します。

```powershell
git pull --rebase origin main
```

その後、もう一度検証します。

```powershell
& 'C:\Users\infop\.cache\codex-runtimes\codex-primary-runtime\dependencies\node\bin\node.exe' scripts\validate-maids.js
```

問題なければ、もう一度pushします。

```powershell
git push origin main
```
