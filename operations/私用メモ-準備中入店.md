# 私用メモ: 準備中メイドさんを追加する手順

`operations/join-preparing.json` を使って、プロフィール未確定のメイドさんを「準備中」として追加する手順です。

## 1. PowerShellで作業場所へ移動

```powershell
cd "C:\Users\infop\OneDrive\ドキュメント\ポトロポータル\potoro-profile-public"
```

## 2. メモ帳でフォーマットを開く

```powershell
notepad operations\join-preparing.json
```

## 3. メモ帳で名前を変更する

この行を探します。

```json
"name": "ここに名前",
```

`ここに名前` を、追加したいメイドさんの名前に変更します。

例:

```json
"name": "ゆあ",
```

必要なら `generation` も変更します。

```json
"generation": "七代目",
```

変更したら、メモ帳で保存して閉じます。

## 4. 反映前チェック

```powershell
& 'C:\Users\infop\.cache\codex-runtimes\codex-primary-runtime\dependencies\node\bin\node.exe' scripts\apply-maid-operation.js operations\join-preparing.json --dry-run
```

成功すると、このように表示されます。

```text
maids.json validation passed (18 entries).
[dry-run] "ゆあ" would be added.
```

## 5. 本反映

```powershell
& 'C:\Users\infop\.cache\codex-runtimes\codex-primary-runtime\dependencies\node\bin\node.exe' scripts\apply-maid-operation.js operations\join-preparing.json
```

## 6. 確認

```powershell
& 'C:\Users\infop\.cache\codex-runtimes\codex-primary-runtime\dependencies\node\bin\node.exe' scripts\validate-maids.js
```

`maids.json validation passed` と表示されればOKです。

## 7. フォーマットを初期状態に戻す

次回また使いやすいように、フォーマットを戻します。

```powershell
& 'C:\Users\infop\.cache\codex-runtimes\codex-primary-runtime\dependencies\node\bin\node.exe' scripts\reset-operation-templates.js
```

## 8. GitHubへ反映

```powershell
git status
git add maids.json operations\join-preparing.json
git commit -m "Add preparing maid profile"
git push origin main
```

手順7でフォーマットを初期状態に戻した場合も、`operations\join-preparing.json` を一緒に `git add` して大丈夫です。
