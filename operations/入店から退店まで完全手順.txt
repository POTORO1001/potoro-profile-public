メイドプロフィール運用 完全手順書
================================

この説明書は、初めて作業する人向けです。
PO・TOROポータル内のメイド紹介ページを更新するための手順を、入店から退店までまとめています。

基本的に使う場所は、次のフォルダです。

C:\Users\infop\OneDrive\ドキュメント\ポトロポータル\potoro-profile-public


とても大事な基本ルール
----------------------

この説明書に出てくるコマンドは、基本的に1行ずつ入力してください。

例えば、次のように4行並んでいる場合でも、

git status
git add maids.json operations\retire.json
git commit -m "Remove maid profile"
git push origin main

慣れるまでは、4行をまとめて貼り付けるのではなく、次のように1行ずつ実行します。

1. git status を入力して Enter を押す
2. 表示された内容を確認する
3. git add ... を入力して Enter を押す
4. 表示された内容を確認する
5. git commit ... を入力して Enter を押す
6. 表示された内容を確認する
7. git push origin main を入力して Enter を押す
8. エラーが出ていないか確認する

理由は、途中でエラーが出た場合に、どこで止まったのか分かりやすくするためです。

特に大事なのは、最後の git push origin main です。
git commit まで終わっていても、git push origin main をしていない場合、GitHubやポータルサイトには反映されません。

つまり、

git commit = 自分のPC内に変更を記録する
git push   = GitHubへ送って、ポータルサイトへ反映させる

という違いがあります。

作業後にサイトへ反映されない場合は、まず git push origin main まで完了しているか確認してください。


最初に必ずやること
------------------

1. PowerShellを開きます。

2. 次のコマンドを貼り付けて、作業場所へ移動します。

cd "C:\Users\infop\OneDrive\ドキュメント\ポトロポータル\potoro-profile-public"

3. GitHub側の最新状態を取り込みます。

git pull --rebase origin main

4. 現在の掲載メンバーを確認したい場合は、次を実行します。

& 'C:\Users\infop\.cache\codex-runtimes\codex-primary-runtime\dependencies\node\bin\node.exe' scripts\list-maids.js


よく使う確認コマンド
--------------------

maids.json が壊れていないか確認するコマンドです。
作業後は必ず実行してください。

& 'C:\Users\infop\.cache\codex-runtimes\codex-primary-runtime\dependencies\node\bin\node.exe' scripts\validate-maids.js

成功すると、次のように表示されます。

maids.json validation passed (〇〇 entries).

この表示が出ればOKです。
エラーが出た場合は、git add / git commit / git push に進まないでください。


GitHubへ反映する基本手順
------------------------

変更作業が終わり、validate-maids が成功したら、次の流れでGitHubへ反映します。
ここも、必ず1行ずつ実行してください。

git status
git add 変更したファイル名
git commit -m "コミットメッセージ"
git push origin main

それぞれの意味は次の通りです。

git status
今の変更状況を確認します。
ここで、変更したファイル名が表示されます。

git add 変更したファイル名
変更したファイルを、次の記録対象に入れます。

git commit -m "コミットメッセージ"
変更内容を、自分のPC内のGitに記録します。
この時点では、まだポータルサイトには反映されません。

git push origin main
自分のPC内に記録した変更をGitHubへ送ります。
この手順まで完了して、初めてポータルサイト側へ反映されます。

例として、maids.json を変更した場合は次のように実行します。
これも1行ずつ実行してください。

git add maids.json
git commit -m "Update maid profile"
git push origin main

git push origin main の後にエラーが出た場合は、まだ反映されていません。
エラー内容を確認してから対応してください。

push後、GitHub Pagesに反映されるまで少し時間がかかることがあります。


1. 入店: プロフィール内容が揃っている場合
-----------------------------------------

この場合は operations/join.json を使います。
名前、誕生日、好きなもの、好きな食べ物、ひとこと、画像まで分かっている時の手順です。

1. 写真を使う場合は、画像ファイルを img フォルダへ入れます。

例:
img/usa.JPG

2. 入店用フォーマットを開きます。

notepad operations\join.json

3. 中身を編集します。

初期状態の例:

{
  "type": "join",
  "maid": {
    "name": "ここに名前",
    "birthday": "1/23",
    "generation": "七代目",
    "likes": [
      "好きなもの1",
      "好きなもの2"
    ],
    "favorite_food": "好きな食べ物",
    "one_liner": "ひとことメッセージ",
    "image": "img/example.jpg"
  }
}

入力例:

{
  "type": "join",
  "maid": {
    "name": "ゆあ",
    "birthday": "8/8",
    "generation": "七代目",
    "likes": [
      "アニメ",
      "甘いもの"
    ],
    "favorite_food": "オムライス、プリン",
    "one_liner": "たくさんお話しに来てください♡",
    "image": "img/yua.JPG"
  }
}

4. メモ帳で保存して閉じます。

5. 反映前チェックをします。

& 'C:\Users\infop\.cache\codex-runtimes\codex-primary-runtime\dependencies\node\bin\node.exe' scripts\apply-maid-operation.js operations\join.json --dry-run

成功例:

maids.json validation passed (20 entries).
[dry-run] "ゆあ" would be added.

6. 問題なければ本反映します。

& 'C:\Users\infop\.cache\codex-runtimes\codex-primary-runtime\dependencies\node\bin\node.exe' scripts\apply-maid-operation.js operations\join.json

7. validate-maids を実行します。

& 'C:\Users\infop\.cache\codex-runtimes\codex-primary-runtime\dependencies\node\bin\node.exe' scripts\validate-maids.js

8. 次回使いやすいように、フォーマットを初期状態に戻します。

& 'C:\Users\infop\.cache\codex-runtimes\codex-primary-runtime\dependencies\node\bin\node.exe' scripts\reset-operation-templates.js

9. GitHubへ反映します。

git status
git add maids.json operations\join.json
git commit -m "Add maid profile"
git push origin main

JSON編集の注意
--------------

1. 最後の項目にカンマを付けない

失敗する例:

"image": "img/usa.JPG",

正しい例:

"image": "img/usa.JPG"

2. 文字列は必ずダブルクォーテーションで囲む

正しい例:

"name": "うさ"

失敗する例:

"name": うさ

3. likes は配列の形にする

正しい例:

"likes": [
  "YouTube",
  "TikTok"
]

4. 画像パスは img/ から始める

正しい例:

"image": "img/usa.JPG"

5. 画像ファイル名の大文字小文字に注意する

img/usa.JPG と img/usa.jpg は別物として扱われます。
実際のファイル名に合わせてください。


よくあるエラーと対応
--------------------

エラー:
operation failed: "うさ" already exists in maids.json.

意味:
すでに maids.json に「うさ」さんがいます。
同じ名前をもう一度追加しようとしている状態です。

対応:
入店追加は不要です。
プロフィール変更をしたい場合は maids.json を編集してください。


エラー:
operation failed: operation file still contains placeholder text.

意味:
operations/join.json や operations/join-preparing.json に「ここに名前」などの仮文字が残っています。

対応:
メモ帳でフォーマットを開き、仮文字を実際の内容に変更してください。


エラー:
maids.json validation failed:
Invalid JSON syntax

意味:
maids.json の書き方が壊れています。
多い原因は、最後の項目に余分なカンマがあることです。

対応:
エラーに表示された line の近くを確認してください。


エラー:
image file not found with exact casing

意味:
maids.json に書いた画像パスと、実際の画像ファイル名が一致していません。

対応:
img フォルダのファイル名と maids.json の image を一致させてください。
大文字小文字も合わせます。


エラー:
git push origin main で fetch first と出る

意味:
GitHub側に、自分のPCにまだ入っていない更新があります。
これは「作業が全部失敗した」という意味ではありません。
GitHub側が先に更新されているため、その更新を自分のPCへ取り込んでから、もう一度 push する必要があるという意味です。

よくある例:
GitHubの画面から画像をアップロードした
別のPCや別の作業で先にGitHubへ反映した
Codexが先にGitHubへpushした

対応:
次のコマンドを1行ずつ実行します。

git pull --rebase origin main

このとき、Fast-forward と表示された場合は、GitHub側の更新をそのまま取り込めたという意味です。
問題ありません。

その後、もう一度 validate-maids を実行します。

& 'C:\Users\infop\.cache\codex-runtimes\codex-primary-runtime\dependencies\node\bin\node.exe' scripts\validate-maids.js

問題なければ、もう一度 push します。

git push origin main

git push origin main が成功すると、次のような表示になります。

main -> main

これが出れば、GitHubへ送信できています。

注意:
git pull --rebase origin main の途中で conflict や CONFLICT と表示された場合は、そこで止めてください。
無理に次のコマンドへ進まず、Codexに表示内容を見せてください。


git status の表示の見方
-----------------------

git status を実行したときに、次のような表示が出ることがあります。

Your branch is up to date with 'origin/main'.

意味:
自分のPCとGitHubが同じ状態です。
作業完了の目安です。


Your branch is ahead of 'origin/main' by 1 commit.

意味:
自分のPCには記録済みの変更がありますが、まだGitHubへ送っていません。

対応:
次を実行します。

git push origin main


Your branch is behind 'origin/main' by 1 commit.

意味:
GitHub側に、自分のPCへまだ入っていない更新があります。

対応:
次を実行します。

git pull --rebase origin main

その後、validate-maids を実行して、問題なければ作業を続けます。


nothing to commit, working tree clean

意味:
未保存のGit変更はありません。
ただし、push が済んでいるかどうかは、上の ahead / behind / up to date の表示も一緒に確認してください。


作業後チェックリスト
--------------------

作業後は、次の4つを確認してください。

1. validate-maids が成功している
2. git status で変更内容を確認した
3. git commit した
4. git push origin main した

最後に次を実行して、

git status

次のように出れば完了です。

On branch main
Your branch is up to date with 'origin/main'.
nothing to commit, working tree clean
